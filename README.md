# 게시판 프로젝트

이 프로젝트는 Spring Boot와 다양한 기술 스택을 활용하여 구현된 게시판 애플리케이션입니다. 사용자 인증, 게시글 작성, 댓글 작성, 데이터베이스 관리 등 다양한 기능을 제공합니다.

## 개발 환경

- **IDE**: IntelliJ IDEA Ultimate
- **JDK**: Java 17
- **빌드 도구**: Gradle 7.4.1
- **프레임워크**: Spring Boot 2.7.0

## 기술 세부 스택

### Spring Boot 기반
- **Spring Boot**: 애플리케이션의 기본 프레임워크
- **Spring Boot Actuator**: 애플리케이션의 모니터링과 관리 기능 제공
- **Spring Web**: 웹 애플리케이션 개발을 위한 필수 모듈
- **Spring Data JPA**: JPA(Java Persistence API)를 통한 데이터베이스 접근
- **Rest Repositories**: RESTful 웹 서비스를 쉽게 구현할 수 있는 리포지토리
- **Rest Repositories HAL Explorer**: HAL 포맷의 RESTful 웹 서비스 탐색 도구
- **Thymeleaf**: 서버사이드 템플릿 엔진으로, HTML을 동적으로 생성
- **Spring Security**: 인증과 권한 부여 기능

### 데이터베이스
- **H2 Database**: 인메모리 데이터베이스로, 개발과 테스트 환경에서 사용
- **MySQL Driver**: MySQL 데이터베이스 연결을 위한 드라이버

### 유틸리티
- **Lombok**: 반복되는 코드를 줄이기 위한 어노테이션 라이브러리
- **Spring Boot DevTools**: 개발 편의성을 위한 도구 모음
- **Spring Configuration Processor**: Spring Boot 설정 파일 자동 완성 지원

### 그 외
- **QueryDSL 5.0.0**: 타입 안전한 SQL 쿼리 생성을 위한 라이브러리
- **Bootstrap 5.2.0-Beta1**: 프론트엔드 스타일링을 위한 CSS 프레임워크
- **Heroku**: 애플리케이션 배포 플랫폼

## 프로젝트 주요 기능

이 프로젝트는 사용자 인증 및 권한 관리, CRUD(생성, 읽기, 업데이트, 삭제) 기능이 포함된 게시판 애플리케이션입니다. 다음과 같은 주요 기능을 제공합니다:

- **사용자 인증**: Spring Security를 사용한 로그인 및 로그아웃 기능.
- **게시글 관리**: 사용자는 게시글을 작성, 수정, 삭제할 수 있으며, 본인이 작성한 글만 관리할 수 있음.
- **댓글 관리**: 게시글에 대한 댓글 작성 및 관리 기능을 제공하며, 본인이 작성한 댓글만 삭제할 수 있음.
- **RESTful API**: Spring Data REST와 HATEOAS를 활용하여 RESTful 서비스를 제공.
- **템플릿 엔진**: Thymeleaf를 사용하여 서버사이드에서 동적으로 HTML 생성.
- **데이터베이스 관리**: H2 데이터베이스를 사용한 개발과 테스트 환경 지원 및 MySQL을 사용한 운영 환경 지원.


## 프로젝트 구조

### ERD(Entity Relationship Diagram)
아래는 프로젝트에서 사용된 데이터베이스의 ERD입니다. 이를 통해 데이터베이스의 구조와 테이블 간의 관계를 쉽게 이해할 수 있습니다.

![ERD](./document/project-board-erd.svg)

1. **회원 (user_account)**:
   - 회원의 기본 정보를 저장하는 테이블로, 사용자 ID, 비밀번호, 이메일, 닉네임 등의 필드를 포함합니다.
   - 회원은 여러 개의 게시글과 댓글을 작성할 수 있으며, 각 회원은 고유한 ID로 식별됩니다.

2. **게시글 (article)**:
   - 회원이 작성한 게시글의 정보를 저장합니다.
   - 제목, 본문, 해시태그, 작성일, 수정일 등을 포함하며, 각 게시글은 하나의 회원에 의해 작성되고, 여러 개의 댓글을 가질 수 있습니다.

3. **댓글 (article_comment)**:
   - 게시글에 달린 댓글의 정보를 저장합니다.
   - 댓글 내용, 작성자, 작성일 등을 포함하며, 특정 게시글에 속하고 하나의 회원에 의해 작성됩니다. 또한, 댓글은 다른 댓글에 대한 답글(대댓글)일 수 있습니다.


### Use Case 다이어그램
아래는 프로젝트의 주요 기능과 사용자가 상호작용하는 방법을 보여주는 Use Case 다이어그램입니다.

![Use Case](./document/use-case.svg)

1. **회원 로그인**:
   - **카카오 인증**: 카카오 계정을 통해 회원 가입 및 로그인.
   - **DB 인증**: 애플리케이션 내에서 직접 관리되는 계정을 통해 로그인.
   - **로그인 실패 처리**: 잘못된 정보로 로그인 시도 시 실패 처리.

2. **게시판 기능**:
   - **게시글 보기**: 사용자는 작성된 게시글을 조회할 수 있습니다.
   - **게시글 쓰기**: 사용자는 새로운 게시글을 작성할 수 있습니다.
   - **게시글 수정**: 사용자는 자신이 작성한 게시글을 수정할 수 있습니다.
   - **게시글 삭제**: 사용자는 자신이 작성한 게시글을 삭제할 수 있습니다.
   - **일반 게시판**: 해시태그가 없는 일반 게시글을 관리하는 게시판입니다.
   - **해시태그 게시판**: 특정 해시태그로 분류된 게시글을 관리하는 게시판입니다.
   - **게시글 검색**: 키워드나 해시태그를 사용해 게시글을 검색할 수 있습니다.

3. **댓글 기능**:
   - **댓글 보기**: 게시글에 달린 댓글을 조회할 수 있습니다.
   - **댓글 쓰기**: 게시글에 댓글을 작성할 수 있습니다.
   - **댓글 삭제**: 사용자는 자신이 작성한 댓글을 삭제할 수 있습니다.

4. **대댓글 기능**:
   - **대댓글 보기**: 댓글에 달린 답글(대댓글)을 조회할 수 있습니다.
   - **대댓글 쓰기**: 댓글에 대댓글을 작성할 수 있습니다.
   - **대댓글 삭제**: 사용자는 자신이 작성한 대댓글을 삭제할 수 있습니다.
